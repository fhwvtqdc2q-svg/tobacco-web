# Repository Instructions

هذه التعليمات مخصصة لمنصة الويب وتقارير TOBACCO/Al-Ameen.

## Review guidelines

- ركز على مشاكل P0/P1: تسريب بيانات، كسر مصادقة، فتح وصول عام، تعطيل اختبار أساسي، أو تغيير يبني/ينشر شيئاً غير مقصود.
- الهدف العملي هو iPhone كمستخدم للواجهة، وWindows كبيئة تطوير وتشغيل محلية. لا تضف هدف macOS ولا تعتبر macOS بديلاً.
- استخدم Windows للبناء أو الفحص عندما توجد أوامر Windows-compatible مثل `npm run check`.
- امنع GitHub Actions أو CI التي تستخدم `macos-*` لبناء المنتج إلا بموافقة صريحة من المستخدم.
- لا يعتبر الاختبار ناجحاً إلا بدليل واضح: أمر تحقق، مخرجات نجاح، أو تأكيد يدوي صريح من المستخدم. لا تستخدم عبارات مثل "يبدو أنه يعمل" كبديل للتحقق.
- الملفات والتقارير الحساسة يجب أن تكون خلف مصادقة: Face ID/biometric على iPhone عندما يكون متاحاً، أو كلمة مرور/Passkey كبديل.
- اسمح بسجلات تدقيق للعمليات الحساسة: وقت العملية، نوع العملية، المستخدم/العميل عند الحاجة، ونتيجة النجاح أو الفشل.
- لا تسجل كلمات المرور الخام، رموز 2FA، مفاتيح API، tokens، cookies، connection strings الحقيقية، أو بيانات دفع كاملة. استخدم redaction أو hash أو آخر 4 أحرف فقط عند الضرورة.
- لا تنشر أو تخدم `.env`, `.git`, `reports` الحساسة، النسخ الاحتياطية، ملفات Excel/CSV الخاصة، أو ملفات config تحتوي أسراراً.
- تحقق من أن أي route أو endpoint أو ملف تقرير حساس محمي بمصادقة وصلاحيات واضحة.
- لا تجعل `reports/`, customer balances, prices, أو sync logs متاحة عبر public static hosting بدون حماية.
- راجع الخصوصية: لا تجمع PII أكثر من اللازم، ولا تحفظ بيانات العملاء في logs بدون سبب عملي واضح.
- راجع الصلاحيات: أقل صلاحية ممكنة، ولا تفتح وصولاً عاماً بـ `0.0.0.0` إلا لحاجة محلية واضحة ومحددة.
- راجع واجهة iPhone: وضوح عربي، قابلية استخدام على شاشة صغيرة، عدم وجود نصوص متداخلة، وعدم الاعتماد على macOS.
- راجع الاعتماديات: لا تضف مكتبة جديدة إلا لحاجة واضحة، ولا تسحب dependency من مصدر غير موثوق.
- راجع Git: لا تعمل reset أو حذف جماعي لتغييرات المستخدم. أي push أو PR يحتاج موافقة صريحة.

## Local verification

- شغل `npm run check` قبل ادعاء نجاح أي تعديل.
- إذا شغلت سيرفر محلي، اذكر عنوان LAN العملي لاختبار iPhone، ولا تدع نجاح iPhone قبل تأكيد الوصول فعلياً.
- لا تدع نجاح deploy أو حماية public files بدون فحص URL أو مخرجات deployment واضحة.
