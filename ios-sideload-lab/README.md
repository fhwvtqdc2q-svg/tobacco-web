# مختبر تعلّم الـ Sideloading وحقن المكتبات على iOS (من ويندوز)

حقيبة بداية تعليمية لفهم بنية تطبيقات iOS، وكيفية حقن مكتبة `.dylib` فيها،
ثم رفع تطبيقك التجريبي إلى الآيفون.

> ⚖️ **للاستخدام القانوني فقط:** طبّق كل ما هنا على **تطبيقك الخاص** أو على
> **تطبيق مفتوح المصدر تملك حق تعديله**. لا تستخدمه لتعديل تطبيقات تجارية
> محمية بحقوق نشر (مثل إنستغرام وغيره).

---

## محتويات الحقيبة

```
ios-sideload-lab/
├── README.md            ← هذا الملف (الدليل المبسّط)
├── setup-ios-lab.ps1    ← سكريبت ويندوز: يهيّئ مجلد ios-lab + يتحقق من Python بضغطة
├── inject_dylib.py      ← سكريبت بايثون: يحقن .dylib داخل .ipa تلقائياً
└── example-tweak/
    ├── Tweak.x          ← كود مكتبة تعليمية (تطبع رسالة عند الحقن)
    └── Makefile         ← يبني Tweak.x إلى MyTweak.dylib عبر Theos
```

---

## الصورة الكبيرة — كيف تعمل العملية كلها؟

```
تطبيقك (.ipa)  +  مكتبتك (.dylib)
        │
        ▼
  inject_dylib.py   ←  (يحقن المكتبة + يعدّل الملف التنفيذي)
        │
        ▼
  MyApp-patched.ipa   ←  (لكنه الآن بلا توقيع صالح)
        │
        ▼
  Sideloadly / AltStore على جهازك   ←  (يعيد التوقيع بحساب Apple ID)
        │
        ▼
      📱 الآيفون
```

**النقطة الأهم:** آخر خطوة (التوقيع + التثبيت على الهاتف) **يجب أن تنفّذها أنت
على جهازك**، لأنها تحتاج كابل USB وحساب Apple ID وجهازك الفعلي. لا يمكن لأي
خادم سحابي القيام بها نيابةً عنك.

## تهيئة بضغطة واحدة على ويندوز (موصى به)

بدل الإعداد اليدوي، شغّل سكريبت التهيئة: انقر بزر الفأرة الأيمن على
`setup-ios-lab.ps1` → **Run with PowerShell**. سيقوم تلقائياً بـ:

- إنشاء مجلد `Desktop\ios-lab\`
- التحقق من Python 3 (واقتراح تثبيته عبر winget إن غاب)
- تنزيل `inject_dylib.py` إلى المجلد
- إنشاء ملف `MyTweak.dylib` تجريبي + طباعة الخطوات التالية

> إن منع ويندوز التشغيل، افتح PowerShell ونفّذ:
> `powershell -ExecutionPolicy Bypass -File .\setup-ios-lab.ps1`

بعدها يبقى عليك فقط وضع `PPSSPP.ipa` في المجلد وتشغيل أمر الحقن.

## الطريق السريع (للمبتدئ تماماً)

إن أردت فقط **تجربة الحقن دون كتابة كود**، فإن **Sideloadly** يفعل كل شيء بنفسه:
حقن الـ dylib + إعادة التوقيع + التثبيت، في خطوة واحدة. اقفز إلى قسم
[تثبيت أول تطبيق](#3-تثبيت-أول-تطبيق-على-الآيفون) واستخدم خانة "Inject dylib" فيه.

السكريبت `inject_dylib.py` هنا **لفهم ما يحدث في الداخل** ولأتمتة الحقن دون
الاعتماد على واجهة Sideloadly — وهو الجزء التعليمي الحقيقي.

## 1) بناء مكتبة `.dylib` التجريبية (Theos على WSL)

على ويندوز، أسهل بيئة لبناء مكتبات iOS هي **Theos** داخل **WSL** (لينكس على ويندوز).

```powershell
# (مرة واحدة) فعّل WSL على ويندوز من PowerShell كمسؤول:
wsl --install
```

ثم **داخل WSL** (أوبنتو):

```bash
# 1) تثبيت Theos (يثبّت أدوات البناء + SDK تلقائياً)
bash -c "$(curl -fsSL https://raw.githubusercontent.com/theos/theos/master/bin/install-theos)"
# أعد فتح الطرفية بعد التثبيت ليُضبط متغير $THEOS

# 2) ادخل مجلد المكتبة وابنِها
cd /mnt/c/.../ios-sideload-lab/example-tweak   # عدّل المسار لمكان مجلدك
make

# الناتج:
#   .theos/obj/MyTweak.dylib
```

> إن لم ترد التعامل مع Theos الآن، تستطيع تجربة `inject_dylib.py` بأي ملف
> `.dylib` جاهز لديك — لكن `MyTweak.dylib` هو مثالنا الذي يطبع رسالة واضحة.

---

## 2) حقن المكتبة داخل الـ IPA

تحتاج فقط **Python 3.8+** (مثبّت على ويندوز من python.org). لا حزم خارجية.

```powershell
# من نافذة CMD أو PowerShell على ويندوز:
cd C:\...\ios-sideload-lab

python inject_dylib.py  MyApp.ipa  MyTweak.dylib
```

سيُنتج `MyApp-patched.ipa`. لتحديد اسم مخصص:

```powershell
python inject_dylib.py  MyApp.ipa  MyTweak.dylib  -o MyApp-modded.ipa
```

ماذا يفعل السكريبت داخلياً؟
1. يفكّ الـ IPA (أرشيف ZIP).
2. ينسخ `MyTweak.dylib` إلى `Payload/MyApp.app/Frameworks/`.
3. يضيف أمر `LC_LOAD_DYLIB` إلى رأس الملف التنفيذي (Mach-O) ليحمّل مكتبتك.
4. يعيد التغليف في IPA جديد.

> **من أين آتي بـ `MyApp.ipa`؟** أنشئ تطبيق "Hello World" خاصاً بك من Xcode
> (إن توفّر Mac لاحقاً)، أو نزّل تطبيقاً **مفتوح المصدر** يوفّر `.ipa` في صفحة
> Releases على GitHub برخصة تسمح بالتعديل.

## 3) تثبيت أول تطبيق على الآيفون

> هذه الخطوة على **جهازك أنت** — تحتاج الآيفون والكابل وحساب Apple ID.

### المتطلبات المشتركة
1. **iTunes** و **iCloud** — حمّلهما **من موقع Apple مباشرة** (وليس من Microsoft Store).
2. **Apple ID** — يُفضّل حساب ثانوي للتجربة.
3. كابل USB + الثقة بالكمبيوتر من الآيفون أول مرة.

> الحساب المجاني: التوقيع صالح **7 أيام** و**3 تطبيقات** كحد أقصى. حساب المطور
> المدفوع (99$/سنة) يرفعها لسنة كاملة.

### الخيار أ — Sideloadly (الأبسط)
1. حمّل من **sideloadly.io** (الرسمي فقط) وثبّته.
2. صل الآيفون → يظهر اسمه في الأعلى.
3. اسحب `MyApp-patched.ipa` (أو الـ IPA الأصلي + ملف dylib عبر خانة "Inject dylib").
4. أدخل بريد Apple ID → اضغط **Start**.
5. على الآيفون: **الإعدادات ← عام ← VPN وإدارة الجهاز** ← ثِق بشهادة المطوّر.
6. افتح التطبيق ✅.

### الخيار ب — AltStore (تجديد تلقائي كل 7 أيام)
1. حمّل **AltServer** من **altstore.io** وثبّته.
2. شغّله (يظهر في شريط المهام) مع تشغيل iTunes و iCloud.
3. من أيقونته: **Install AltStore ← [جهازك]** ← أدخل Apple ID.
4. ثِق بالشهادة من **إدارة الجهاز**.
5. افتح **AltStore** على الآيفون ← تبويب **My Apps** ← **+** ← اختر الـ IPA.

## التحقّق من نجاح الحقن

بعد فتح التطبيق على الآيفون، افتح سجل الجهاز (Console.app على ماك، أو أدوات
عرض السجلّات) وابحث عن:

```
[MyTweak] 🎉 تم حقن المكتبة بنجاح!
[MyTweak] التطبيق أصبح نشطاً الآن ✅
```

ظهور هذه الرسائل = نجح حقن مكتبتك وتُحمَّل مع التطبيق.

## استكشاف الأخطاء

| المشكلة | السبب المحتمل | الحل |
|---|---|---|
| `لا توجد مساحة كافية في رأس الملف` | الملف التنفيذي بلا حشو كافٍ لأمر جديد | استخدم خانة Inject الخاصة بـ Sideloadly (تتعامل مع هذه الحالة) |
| التطبيق يفتح ثم يُغلق فوراً | توقيع غير صالح، أو معمارية dylib لا تطابق الجهاز | أعد التوقيع عبر Sideloadly؛ تأكد أن الـ dylib مبنية لـ arm64 |
| لا تظهر رسائل `[MyTweak]` | المكتبة لم تُحمّل | تحقق أن اسم الملف في Frameworks/ يطابق `LC_LOAD_DYLIB` |
| `python` غير معروف على ويندوز | Python غير مثبّت/غير مضاف لـ PATH | ثبّت من python.org وفعّل "Add to PATH" |
```
